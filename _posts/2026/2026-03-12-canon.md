[PageLeft](/copyleft-pagerank) indexes 98 pages. All mine. I search it and get my own writing back. A copyleft search engine with one author is a diary with a ranking algorithm.

The architecture works. Embeddings find meaning across vocabulary. The copyleft filter keeps the index honest. PageRank surfaces what peers value. But peers require more than one person. The search engine needs a body of work that doesn't exist yet. Most of the world's code is proprietary because that's convention, not because it's better. We already [proved](/paradox-of-open-competition) that being open wins. The canon is empty because nobody's tried filling it, not because the model is wrong.

A coding agent helped me write a blog post. The blog post got published under [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/). Then the agent read the post and [built the thing](https://github.com/kimjune01/pageleft). The blog post compiled to code. The [reference implementation](https://github.com/kimjune01/pageleft) drew from [another copyleft codebase](/vector-space) for its embedding pipeline and sidecar pattern. Copyleft is transitive. The new code inherits the obligation: share alike, or don't ship. Even if a stranger had built PageLeft from the same blog post, their implementation would carry the same license forward. That's the point.

A blog post that describes an architecture precisely enough for a coding agent to implement it one-shot is different from one that hand-waves. The compilable posts are denser, more useful. They're specifications that happen to read like prose.

Any compilable blog post can cite the entire body of copyleft work as reference implementation. No licensing fear. No negotiation. The [VectorSpace adserver](/vector-space), the [PageLeft crawler](https://github.com/kimjune01/pageleft), every copyleft repo on the internet: all of it is fair game for the next blog post that compiles to code. And when that code ships, it joins the canon. The canon grows. The next writer has more to draw from. The next agent has more reference implementations to pattern-match against.

PageLeft already indexes by semantics. A Go project on Codeberg, a Python notebook on someone's Jekyll site, a Rust crate with an AGPL header: if it's copyleft and it's reachable, it's searchable. The canon isn't limited to what one person wrote. It just needs more people to write.

The workflow: write a blog post. Hand it to a coding agent. See if it compiles. If it doesn't, tighten the spec — one pass. Publish copyleft. The agent fills in every gap between intent and implementation. The human's job is to be clear about what they want. The machine's job is to make it real. The license's job is to keep it free.

Ninety-eight pages is lonely. But as more companies discover what open competition already shows, they'll open their code. Every compilable copyleft post is a brick. The wall gets taller. The next builder stands on everything that came before. The canon fills itself.
